# hermes-overnight

Long-term research-dump repo for autonomous Hermes Agent product-intelligence runs.

Anything produced by an overnight / unattended research pass lives here, organized by topic. No CI, no schema — just an archive you can grep.

## Contents

| folder | contains | origin |
|---|---|---|
| `dayo-intel/` | 5 docs + README for the Dayo product-intelligence mission (Dayo PWA, Indian engineering students). Includes competitor analysis, student pain points, V2 feature spec, UX copy, GTM + launch copy. | overnight Dayo intelligence run |
| `opus-leads/` | Lead generation + cold DM + follow-up sequences for Opus agency outreach. | overnight Opus sales run |
| `opus-sales-kit/` | Case study, followup templates, objections, pricing, proposal template. | overnight Opus sales run |

## Dayo — what to read in what order

1. `dayo-intel/competitor_analysis.md` — competitive landscape + Dayo's moat
2. `dayo-intel/student_pain_points.md` — what students actually complain about
3. `dayo-intel/dayo_v2_spec.md` — what to build next
4. `dayo-intel/dayo_ux_copy.md` — every UI string for V1, ready to drop in
5. `dayo-intel/dayo_launch_kit.md` — how to take it public

A canonical public mirror of the Dayo materials lives at https://github.com/25sidhx/dayo-product-research — read either location, they are byte-identical.

## Conventions

- Markdown only. No code lives here.
- Each subdirectory carries its own README as the entry point.
- File naming uses snake_case / hyphens, no spaces.
- This repo is public so anyone can clone it as a reference; do not put private research (real client data, PII) in here.
